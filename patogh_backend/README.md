# بک‌اند جنگو برای «بوک پاتوق»

این پروژه، دیتابیس واقعی (Django + SQLite) و یک REST API کامل برای فایل
`book_patogh.html` می‌سازد؛ همان اپی که فعلاً همه‌چیز را در `localStorage`
مرورگر نگه می‌داشت.

## ساختار

| اپ          | مدل‌ها                     | توضیح                                        |
|-------------|----------------------------|-----------------------------------------------|
| `catalog`   | `Author`, `Book`           | کتاب‌ها و نویسندگان                           |
| `blog`      | `Post`                     | پست‌های وبلاگ                                 |
| `inquiries` | `BookRequest`              | فرم «درخواست‌ها»                              |
| `orders`    | `Order`, `OrderItem`       | سفارش‌ها (نتیجه‌ی تسویه‌حساب)                  |
| `accounts`  | (از `User` خودِ جنگو استفاده می‌شود) | ثبت‌نام / ورود مشتری و ادمین |

هر رکورد یک `id` رشته‌ای کوتاه می‌گیرد (مثلاً `idab12xyz`)، دقیقاً هم‌شکل
همان چیزی که تابع `uid()` در جاوااسکریپت اصلی می‌ساخت — بنابراین منطق مقایسه‌ی
آی‌دی‌ها در فرانت‌اند (`b.id === id`) بدون تغییر کار می‌کند.

## راه‌اندازی

```bash
cd patogh_backend
python -m venv venv && source venv/bin/activate      # ویندوز: venv\Scripts\activate
pip install -r requirements.txt

python manage.py makemigrations catalog blog inquiries orders accounts
python manage.py migrate
python manage.py seed_data          # همان ۶ کتاب و ۲ پست پیش‌فرض قبلی را می‌ریزد
python manage.py createsuperuser    # حساب ادمین فروشگاه را همینجا بساز (برای ورود به پنل مدیریت سایت)

python manage.py runserver
```

سرور روی `http://127.0.0.1:8000` بالا می‌آید. پنل واقعی ادمین جنگو هم روی
`/admin/` در دسترس است (کامل‌تر از پنل قبلی HTML، و می‌توانی بجای/کنار API
مستقیماً از همینجا کتاب/سفارش/... مدیریت کنی).

## نقاط پایانی API

همه چیز زیر `/api/` است و JSON برمی‌گرداند:

| متد | مسیر | دسترسی | توضیح |
|---|---|---|---|
| GET | `/api/books/` | همه | لیست کتاب‌ها |
| POST | `/api/books/` | ادمین | افزودن کتاب |
| GET/PUT/DELETE | `/api/books/{id}/` | ادمین برای ویرایش/حذف | جزئیات/ویرایش/حذف |
| GET/POST/PUT/DELETE | `/api/authors/` , `/api/authors/{id}/` | مثل بالا | نویسندگان |
| GET/POST/PUT/DELETE | `/api/posts/` , `/api/posts/{id}/` | مثل بالا | وبلاگ |
| POST | `/api/requests/` | همه | ثبت درخواست جدید |
| GET/PUT/DELETE | `/api/requests/{id}/` | ادمین | مدیریت درخواست‌ها |
| POST | `/api/orders/` | همه (مهمان هم می‌تواند) | ثبت سفارش نهایی |
| GET/PUT/DELETE | `/api/orders/{id}/` | ادمین | مدیریت سفارش‌ها |
| POST | `/api/auth/signup/` | همه | ثبت‌نام مشتری |
| POST | `/api/auth/login/` | همه | ورود (مشتری *و* ادمین) |
| POST | `/api/auth/logout/` | کاربر واردشده | خروج |
| GET | `/api/auth/me/` | کاربر واردشده | اطلاعات کاربر جاری |

نوشتن (POST/PUT/DELETE) روی کتاب/نویسنده/وبلاگ/درخواست‌ها/سفارش‌ها نیاز به
هدر `Authorization: Token <token>` دارد که از `login/` می‌گیری، و کاربر باید
`is_staff=True` باشد (همان که با `createsuperuser` ساختی). یعنی رمز مشترکِ
پنل ادمینِ قدیمی، الان یک حساب کاربری واقعی و امن‌تر شده.

## وصل‌کردن به همین فایل HTML موجود

خبر خوب: در کد جاوااسکریپت اصلی، تمام رفت‌وآمد با داده در چند تابع مشخص و
جمع‌وجور تعریف شده (`loadData`, `saveBook`, `deleteBookRemote`, `saveAuthor`,
`saveRequest`, `saveOrder`, `submitAuth`, ...). کافیست فقط همین توابع را با
نسخه‌ای که به این API وصل می‌شود جایگزین کنی؛ بقیه‌ی صفحه (رندر کارت کتاب،
روتینگ، فیلتر، سبد خرید و ...) دست‌نخورده باقی می‌ماند چون همه از روی آبجکت
`DATA` که در حافظه است کار می‌کنند.

فایل زیر (`frontend-api-adapter.js`) این توابع را بازنویسی کرده؛ محتوایش را
داخل تگ `<script>` فایل HTML، **جایگزین** توابع هم‌نامشان کن (یا کل بلوک را
درست بعد از `let DATA = {...}` بچسبان و نسخه‌های قدیمی‌شان را حذف کن).

نکات ریز لازم برای هماهنگی کامل:
1. تاریخ پست‌ها الان `date` را به‌صورت ISO برمی‌گرداند، نه رشته‌ی فارسی
   آماده. هرجا `esc(p.date)` چاپ می‌شود را به
   `new Date(p.date).toLocaleDateString('fa-IR')` تغییر بده.
2. در `renderCheckoutSuccess`، چون الان شماره سفارش را خودِ بک‌اند می‌سازد
   (نه `Math.random()` در مرورگر)، باید بعد از `await saveOrder(...)` مقدار
   `orderNo` برگشتی از سرور را نمایش بدهی (در آداپتور پایین انجام شده).
3. مقدار `gateway` که برای سرور فرستاده می‌شود باید `id` درگاه باشد
   (مثلاً `'zarinpal'`)، نه برچسب فارسی‌اش؛ در آداپتور همین‌طور اصلاح شده.
